Introduction_to_Unity

Unity is a cross-platform game engine used to develop 2D, 3D, augmented reality (AR), and virtual reality (VR) experiences.

It supports deployment to multiple platforms including mobile, PC, consoles, and the web.

UnityEditor.EditorApplication.isPlaying = false;
GameObjects_and_Components

Everything in a Unity scene is a GameObject. Components add functionality to these GameObjects.

For example, a Rigidbody component allows a GameObject to obey the laws of physics.

  • Transform
  • Mesh Renderer
  • Collider
  • Rigidbody
  • Audio Source
gameObject.AddComponent();
Scenes_and_Prefabs

Scenes are individual levels or screens in a Unity game. Prefabs are reusable GameObject templates.

Using prefabs, you can maintain consistency and improve workflow efficiency.

Instantiate(enemyPrefab, transform.position, transform.rotation);
Scripting_with_C_Sharp

C# is the primary language used in Unity. Scripts define behavior for GameObjects.

MonoBehaviour is the base class from which every Unity script derives.

void Start() { Debug.Log("Game Started"); }

The Update method is called once per frame and is used to implement behaviors over time.

void Update() { transform.Translate(Vector3.forward * Time.deltaTime); }
Physics_and_Collisions

Unity provides a powerful physics engine with built-in support for 2D and 3D collisions.

Rigidbodies and Colliders are essential for physics-based interactions.

  • Box Collider
  • Sphere Collider
  • Capsule Collider
  • Mesh Collider
  • Character Controller
void OnCollisionEnter(Collision collision) { Debug.Log("Collision Detected with " + collision.gameObject.name); }
